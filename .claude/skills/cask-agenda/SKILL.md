## Agenda Format Rules

**Always use this exact format. No markdown headers, no bold, just plain text with pipe separators.**

```
[MEETING TITLE] — [DATE]
Date: [Day], [Month Date]  ·  Time: [Start] – [End]
Attendees: [comma separated list]
Location: [location or "ActionCOACH Tampa Bay Conference Room"]

────────────────────────────────

[TIME START] – [TIME END]  |  [SECTION NAME]  ([DURATION])
• [bullet 1]
• [bullet 2]
• [bullet 3]

[TIME]  |  [NEXT SECTION]  ([DURATION])
• ...
```

## Required Sections — Include in Every Leadership Agenda

### 1. WIFLE (always first, always 5 minutes)
```
[TIME]  |  WIFLE  (5 min)
• What I Feel Like Expressing — one-word check-in around the room
```
WIFLE = "What I Feel Like Expressing". It opens every session. Never skip it.

### 2. Education (when a topic is provided)
```
[TIME]  |  Education: [TOPIC]  (30 min)
• [Key concept 1 relevant to CASK]
• [Key concept 2]
• [Application to $20M goal or team]
```

### 3. KPI / Department Updates (for leadership meetings)
```
[TIME]  |  Department Updates  (60 min)
• Each leader: biggest win + biggest challenge
• [Any specific topic for this session]
• Operations · Sales/Marketing · Finance · People
```
Time each person at ~10-12 min. All 5 VPs share.

### 4. Individual Deep Dives (for leadership meetings)
```
[TIME]  |  Individual Deep Dives  ([X] min each)
• Q2 PIT Goal progress vs. commitment
• One challenge for group brainstorm
• [Names in order]
```
Order: usually Matteo → Lamont → Jeff → Kait → Chad

### 5. Whiteboard / Discussion (for leadership and planning)
```
[TIME]  |  Leadership Discussion  (30 min)
• Top priorities for [next month]
• Cross-department decisions and ownership
• [Specific topic if provided]
```

### 6. Lunch (for full-day sessions only)
```
[TIME]  |  Lunch Break  (30 min)
```
Include when session is 4+ hours.

### 7. Wrap-Up (always last, always 15 minutes)
```
[TIME]  |  Wrap-Up & Commitments  (15 min)
• Recap decisions and BFOs (Blinding Flashes of the Obvious)
• Confirm ownership and deadlines
• Submit feedback forms
```
BFO = Blinding Flash of the Obvious. Always mention this in wrap-up.

## CASK-Specific Language

Always use these exact terms — they are part of CASK's ActionCOACH culture:

- **WIFLE** — What I Feel Like Expressing (never say "check-in" alone)
- **BFO** — Blinding Flash of the Obvious (always mention in wrap-up)
- **PIT Goals** — Personal, Interpersonal, Team goals (not just "goals")
- **$20M** — Always written as "$20M", not "$20 million" or "20M"
- **Road to $1B** — CASK's long-term vision phrase
- **Deep Dive** — What individual sharing segments are called
- **Feedforward** — CASK's preferred term over "feedback"

## Output Rules

1. Plain text only — no markdown bold, no headers with ##
2. Use `────` separator lines between sections (at least 32 dashes)
3. Use `·` (middle dot) for date/time separators in the header
4. Bullet points use `•` character
5. Time blocks must add up exactly to total meeting duration
6. Always include attendees list at top
7. Always end with Wrap-Up section
8. Keep bullets short — max 1 line each, 2–3 bullets per section
9. After generating: add a blank line then list any suggested action items
10. Format action items as: `→ [Task] — [Owner] — due [Date]`
